### Decisiones técnicas
- Socket Unix → TCP: ollama 0.24.0 parsea mal `OLLAMA_HOST=unix:///run/ollama/ollama.sock`. Se revierte a TCP con bind a 127.0.0.1.
- `acquire_boot_vram` bypassa la cola (reserva directa en `_active`) para evitar deadlock en boot.
- Proxy gateway admite GET/POST a `/api/*` con control de admisión por VRAM.
- `ura-llama-server.service` con `After=ura-mochila.service` para asegurar que mochila esté lista antes del pre-check VRAM.
- Stubs creados con `chattr +i` para que `git` no los revierta.

### Fase Aislamiento: Sandbox Docker para pre-commit

**Objetivo:** Ejecutar análisis dinámico del código generado en contenedor efímero antes de tocar el filesystem real.

#### Arquitectura

```
sandbox/
  ├── Dockerfile              # python3.11 + bash + yamllint + requirements
  ├── run_test.py             # script entrypoint que recibe path + type
  └── test_runner.sh          # wrapper bash para multi-lenguaje
```

#### Flujo Sandbox

```
1. safe_write() escribe a /tmp/ura_sandbox/<hash>.py
2. docker run --rm --network none \
     -v /tmp/ura_sandbox/<hash>.py:/app/code.py \
     ura-sandbox:latest \
     python3 /app/run_test.py /app/code.py py
3. parsear stdout/stderr del contenedor
4. exit code 0 → commit. exit code != 0 → rollback.
```

#### Tipos de test por extensión

| Extensión | Test | Comando |
|---|---|---|
| `.py` | Import + syntax + type hints | `python3 -c "import ast; ast.parse(open('/app/code.py').read())" && python3 -c "import sys; sys.path.insert(0,'/app'); import code"` |
| `.sh` | Syntax check | `bash -n /app/code.py` |
| `.json` | JSON válido + schema opcional | `python3 -c "import json,sys; json.load(open('/app/code.py'))"` |
| `.yaml` | YAML válido | `python3 -c "import yaml,sys; yaml.safe_load(open('/app/code.py'))"` |

#### Consideraciones de infraestructura

| Aspecto | Decisión |
|---|---|
| **Red** | `--network none` — el sandbox NO debe alcanzar ollama/mochila/Qdrant |
| **GPU** | `--gpus none` — el sandbox no necesita CUDA. Si el import requiere torch, debe saltar con `ImportError` manejado |
| **Timeout** | `docker run --timeout 10` (Docker 29+ soporta `--timeout` nativo) |
| **Imagen** | `ura-sandbox:latest` construida una vez, cacheada localmente |
| **Volumen** | Solo montar el archivo bajo test, NO todo el repo |
| **Logs** | stdout/stderr del contenedor → logging estructurado de mochila |

#### Dockerfile de referencia

```dockerfile
FROM python:3.11-slim-bookworm
RUN apt-get update && apt-get install -y --no-install-recommends \
    bash yamllint && rm -rf /var/lib/apt/lists/*
COPY run_test.py /app/run_test.py
ENTRYPOINT ["python3", "/app/run_test.py"]
```

#### Dependencias externas
- Docker socket montado en mochila (o acceso vía `docker-py`)
- Imagen base `python:3.11-slim-bookworm` (~120MB) disponible en ARM64
- `yamllint` para validación YAML avanzada

#### Archivos a crear
- `sandbox/Dockerfile`
- `sandbox/run_test.py`
- `sandbox/test_runner.sh`
