## Goal

Integrate KE compile → FusionPipeline → SemanticMemory + HistoricalMemory into production, connect `motor/assistant/` to that flow, then instrument assistant with observability using existing infra.

## Constraints & Preferences (V6.1 Rule 0)

- Prohibited: new engines, pipelines, stages, architectural layers
- Prohibited: rewriting existing components unless evidence of a functional defect
- Prohibited: creating new files if existing ones can be extended (Rule 1)
- Each phase must end with a working end-to-end flow + test (Rule 2)
- Post-phase audit: consumers, execution %, orphaned code, duplicates (Rule 3)
- ARQ-800: before any data transformation, document source owner, target owner, conversion location, existing equivalents, and standard status
- Subscriber must be thin (~12 lines) — delegate orchestration to KE and Fusion, not absorb it
- If audit reveals a better architectural solution than planned, stop and propose it before implementing
- Fase solo puede cerrarse cuando exista evidencia de que la funcionalidad se ejecuta realmente en producción, tiene consumidores, aparece en telemetría y está completamente integrada
- F3 must NOT create a third observability system — reuse `motor/observability/` for metrics, health, logging, correlation_id
- All conclusions must be based on evidence (grep output with file:line), not inference

## Architectural Decision (Alternative B)

Three alternatives were evaluated:

| Alternative | Description | Verdict |
|---|---|---|
| A | `run_on_compile` callback in Fusion | ❌ Fusion would depend on KE |
| B | KE EventBus subscriber triggers Fusion | ✅ Chosen |
| C | Composition root wires both | ❌ Adds unnecessary layer |

**Decision:** Alternative B — Fusion subscribes to KE EventBus. KE publishes `CompileCompleted`, fusion subscriber reacts.

## Key Architectural Decisions

| Decision | Rationale | Evidence |
|---|---|---|
| Alternative B (KE subscriber) | Fusion does not import KE; KE does not know about Fusion | `subscribers.py:_make_fusion_subscriber()` |
| Thin subscriber | ~15 lines, delegates to KE.compile_result_to_claims() + Fusion.run_fusion_on_claims() | `subscribers.py:176-197` |
| ARQ-800: KE owns claim conversion | `orchestrator.py` produces KnowledgeClaim; Fusion consumes | `compile_result_to_claims()` in orchestrator |
| Idempotent subscribe_all | `_SUBSCRIBED` flag prevents double registration | `subscribers.py:17-20` |
| Lazy collector init | `_BUS_INITIALIZED` guard in collector.py | `collector.py:7-8` |
| Output dict compatible | `run_fusion_on_claims` produces dicts compatible with `SemanticFact(**dict)` | F2 bridge to SemanticMemory |
