# Session 2026-06-12

## Goal
Stabilize GX10 (ASUS) after repeated freezes caused by unified memory exhaustion from 4 llama.cpp servers + 7 orphan systemd services.

## Trigger
System froze overnight. OpenCode unreachable. User rebooted at 12:47.

## Diagnosis
- 91 GB / 121 GB used (75%), 1.4 GB swap, memory pressure avg300=1.64
- 4 llama.cpp servers running forever (no TTL): codestral-22b, qwen2.5-coder-q8_0, qwen2.5-coder-32b, qwen2-vl-7b
- Ollama: only 27 MB RSS, 0 models loaded (KEEP_ALIVE=1m works)
- model-router killed 5x with SIGKILL (signal 9) by kernel under memory pressure
- Cascading failures: router dead -> central-router -> health checks -> all failed
- 7 systemd services failed at boot (scripts deleted but unit files remained)
- netfilter-persistent failed at boot (cgroup rule in rules.v4 couldn't restore)

## User corrections to original plan
- Ollama was wrongly assumed as the memory hog; real cause was 4 llama.cpp servers
- nvtop won't work on GB10 (unified memory, nvidia-smi returns N/A)
- MemoryHigh vs MemoryMax: correct distinction (soft vs hard limit)
- TTL needs to apply to llama.cpp too, not just Ollama
